## Fixes Applied

### 1. `keras_hub/src/utils/litertlm/adapter.py`

**Problem**: Gemma4's vision encoder expects pre-processed patches (`pixel_values` + `pixel_position_ids`), not raw RGB images. The adapter's `forward_prefill` only accepted `images`.

**Fix**: Added `pixel_values` and `pixel_position_ids` parameters. Detect Gemma4-style vision encoder by checking if `vision_encoder.inputs` has exactly 2 inputs named `pixel_values` and `pixel_position_ids`. This avoids heavy imports (e.g., checking for `Gemma4VisionEncoder`).

### 2. `keras_hub/src/utils/litertlm/export.py`

**Problem A**: `_build_vision_sample_inputs` created raw RGB images, which fail for Gemma4.

**Fix A**: Added `_build_gemma4_vision_sample_inputs` that creates:
- `pixel_values`: shape `(B, max_images, num_patches, patch_dim)` (float32)
- `pixel_position_ids`: shape `(B, max_images, num_patches, 2)` (int32)

Also updated `export_to_litertlm` to detect Gemma4 vision encoder (same heuristic as adapter) and call the correct builder.

**Problem B**: `_build_audio_sample_inputs` hardcoded `128` mel bins, but Gemma4 audio encoder uses `64`.

**Fix B**: `_get_audio_config` now extracts `audio_input_feat_size` from the preprocessor (defaults to 128). `_build_audio_sample_inputs` accepts this value.

### 3. `_PrefillAdapter.forward` in `export.py`

Updated to forward `pixel_values` and `pixel_position_ids` through to the underlying adapter.

## Known Issues & Blockers

### 1. Image Preprocessing in Export Graph

`Gemma4ImageConverter` uses `ops.image.resize` with `bicubic` + `antialias=True`. This lowers to `aten._upsample_bicubic2d_aa.default`, which is **not supported** by `litert_torch` converter.

**Workaround**: Export accepts pre-processed `pixel_values`/`pixel_position_ids`. The Android app (or caller) must do image preprocessing externally before calling the model.

### 2. Android Emulator — No KVM

This environment lacks `/dev/kvm`. The emulator cannot start even with `-no-accel`.

**Workaround**: Test on:
- Physical ARM64 Android device
- Firebase Test Lab
- Local machine with KVM enabled

### 3. OOM Risk with Actual Weights

Gemma4 2B is ~10 GB in bfloat16. Ensure device/VM has sufficient RAM before loading full weights.
